npm 的命令主要分为**项目管理**、**包操作**、**配置与安全**、**发布与维护**等几大类。这里帮你梳理了一份完整的命令清单，同时标注了常用的参数和简写。

### 📌 项目管理类

| 命令 | 常用参数 / 简写 | 说明 |
| :--- | :--- | :--- |
| **`npm init`** | `-y` 或 `--yes` | 在当前目录生成 `package.json` 文件。使用 `-y` 可跳过向导，快速生成。 |
| **`npm run`** | 无 | 运行 `package.json` 中 `"scripts"` 字段定义的脚本，如 `npm run dev`。 |
| **`npm test`** | 无 | 运行测试脚本，是 `npm run test` 的简写。 |
| **`npm start`** | 无 | 启动项目，通常用于执行 `node server.js` 等命令，是 `npm run start` 的简写。 |
| **`npm version`** | `patch`, `minor`, `major` | 更新包的版本号。例如 `patch` 对应修订号（bug修复），`minor` 对应次版本号（新功能）。 |

### 📦 包操作类

| 命令 | 常用参数 / 简写 | 说明 |
| :--- | :--- | :--- |
| **`npm install`** | `i` (简写), `-g` (全局), `-D` (开发依赖), `-S` (生产依赖) | 根据 `package.json` 安装所有依赖。`npm i lodash` 可安装指定包。 |
| **`npm uninstall`** | `rm` (简写), `-g`, `-D`, `-S` | 卸载包，并从 `package.json` 中移除相应依赖。 |
| **`npm update`** | 无 | 依据版本规则，将包更新到最新兼容版本。 |
| **`npm list`** | `ls` (简写), `-g` (全局), `--depth=0` (层级) | 以树形结构列出所有已安装的包及其依赖关系。 |
| **`npm ci`** | 无 | 基于 `package-lock.json` 进行**快速、干净的安装**，常用于CI/CD环境，要求必须存在锁文件。 |
| **`npm link`** | 无 | 将本地包链接到全局，用于本地调试未发布的npm包。 |
| **`npm cache`** | `clean --force`, `verify` | 管理npm的本地缓存。`verify` 用于校验缓存完整性。 |
| **`npm dedupe`** | 无 | 整理依赖树，将多个版本的依赖尽可能合并，减少磁盘占用。 |
| **`npm prune`** | 无 | 移除 `node_modules` 中未被 `package.json` 声明的多余包。 |

### 🔧 配置与安全类

| 命令 | 常用参数 / 简写 | 说明 |
| :--- | :--- | :--- |
| **`npm config`** | `list`, `get`, `set`, `delete` | 管理npm配置（如 registry 镜像源）。例如 `npm config set registry <url>`。 |
| **`npm audit`** | `fix`, `--force` | 检查项目依赖中的安全漏洞。`npm audit fix` 可尝试自动修复。 |
| **`npm outdated`** | 无 | 检查当前项目中有哪些依赖包版本已过时。 |
| **`npm doctor`** | 无 | 检查本地npm环境的基本健康状况（如网络、权限、缓存等）。 |
| **`npm explain`** | 无 | 解释为什么某个包被安装，展示其依赖关系链。 |

### 🚀 发布与维护类

| 命令 | 常用参数 / 简写 | 说明 |
| :--- | :--- | :--- |
| **`npm login`** | `adduser` (注册) | 登录npm registry账号。 |
| **`npm logout`** | 无 | 退出当前登录的registry账号。 |
| **`npm publish** | `--access public/restricted` | 将当前包发布到npm仓库。 |
| **`npm unpublish`** | `--force` | 从registry撤销已发布的包版本（有严格限制，通常72小时内可撤）。 |
| **`npm pack`** | 无 | 将包打包成一个 `.tgz` 文件，方便查看发布内容。 |
| **`npm deprecate`** | 无 | 为某个包版本添加弃用警告，安装时会显示。 |
| **`npm dist-tag`** | `add`, `rm`, `ls` | 管理包的发布标签（如 `latest`, `beta`）。 |
| **`npm owner`** | `add`, `rm`, `ls` | 管理npm包的所有者权限。 |
| **`npm whoami`** | 无 | 显示当前登录的用户名。 |

### 🛠️ 其他实用命令

| 命令 | 常用参数 / 简写 | 说明 |
| :--- | :--- | :--- |
| **`npm help`** | 无 | 在浏览器中打开npm的官方文档。 |
| **`npm search`** | 无 | 在npm仓库中搜索包。 |
| **`npm view`** | `v` (简写) | 查看包的registry信息（如版本历史、依赖等）。 |
| **`npm root`** | `-g` | 显示当前项目中 `node_modules` 的路径。 |
| **`npm prefix`** | `-g` | 显示当前目录或全局的前缀路径。 |
| **`npm exec`** | `npx` (更常用) | 从本地或远程包中运行命令，`npx` 是其更便捷的替代品。 |
| **`npm sbom`** | 无 | 生成当前项目的软件物料清单（SBOM）。 |
| **`npm fund`** | 无 | 列出项目依赖包的资助信息，用于支持开源作者。 |
| **`npm repo`** | 无 | 在浏览器中打开包的源代码仓库主页。 |
| **`npm docs`** | 无 | 在浏览器中打开包的文档主页。 |

你可以通过 `npm -l` 在终端看到大部分命令的简单说明，`npm help <command>` 则可以查看某个具体命令的详细用法。

希望这份清单能帮你更全面地了解 npm。如果想深入了解某个命令的具体用法，随时可以再问我。

你提到的很关键，除了 `-g`、`-D`、`-S` 等简写参数，npm 还有很多**全局通用参数**和**命令特有参数**。它们能精细控制安装行为、网络、输出等。

下面是整理好的核心参数清单，以及常用场景示例：

### 📌 常用全局参数（适用于大多数命令）

| 参数 / 简写 | 作用 | 示例 |
| :--- | :--- | :--- |
| **`--save` / `-S`** | 安装/卸载时写入 `dependencies`（生产依赖） | `npm i lodash -S` |
| **`--save-dev` / `-D`** | 写入 `devDependencies`（开发依赖） | `npm i jest -D` |
| **`--global` / `-g`** | 全局安装/卸载/查看 | `npm i -g npm` |
| **`--save-prod`** | 强制写入 `dependencies`（即使有 `--production` 环境变量） | `npm i --save-prod pkg` |
| **`--save-optional` / `-O`** | 写入 `optionalDependencies`（可选依赖） | `npm i -O pkg` |
| **`--save-exact` / `-E`** | 安装精确版本（不添加 `^` 前缀） | `npm i -E lodash@4.17.21` |
| **`--save-bundle` / `-B`** | 同时将包加入 `bundleDependencies` | `npm i -B pkg` |

### 🌐 网络与注册表相关

| 参数 | 作用 | 示例 |
| :--- | :--- | :--- |
| **`--registry <url>`** | 临时指定包的下载源 | `npm i --registry=https://registry.npmmirror.com` |
| **`--proxy <url>`** | 设置 HTTP 代理 | `npm config set proxy http://proxy.company.com:8080` |
| **`--no-proxy`** | 忽略代理环境变量 | `npm i --no-proxy` |
| **`--https-proxy <url>`** | 设置 HTTPS 代理 | 类似 `--proxy` |
| **`--strict-ssl`** | 是否严格校验 SSL 证书（`false` 可忽略自签名证书） | `npm config set strict-ssl false` |
| **`--cafile <path>`** | 指定自定义 CA 证书文件 | `npm i --cafile ./my-ca.crt` |

### ⚙️ 安装与执行行为控制

| 参数 | 作用 | 示例 |
| :--- | :--- | :--- |
| **`--production`** | 只安装 `dependencies`，跳过 `devDependencies` | `npm ci --production` |
| **`--only={prod|dev}`** | 同上，更明确 | `npm i --only=prod` |
| **`--no-save`** | 安装但不写入 `package.json` | `npm i lodash --no-save` |
| **`--dry-run`** | 模拟执行，显示会做什么但不实际安装 | `npm install --dry-run` |
| **`--ignore-scripts`** | 禁止执行任何 `package.json` 中的 `preinstall`、`postinstall` 等脚本 | `npm i --ignore-scripts` |
| **`--force`** | 覆盖冲突、重新构建等（谨慎使用） | `npm i --force` |
| **`--legacy-peer-deps`** | 忽略 peerDependency 冲突（npm 7+ 默认严格） | `npm i --legacy-peer-deps` |
| **`--strict-peer-deps`** | 遇 peerDependency 冲突直接失败 | `npm i --strict-peer-deps` |
| **`--no-package-lock`** | 不生成或忽略 `package-lock.json` | `npm i --no-package-lock` |
| **`--package-lock-only`** | 只更新 `package-lock.json`，不实际安装 | `npm i --package-lock-only` |
| **`--no-audit`** | 安装时不自动执行 `audit` 检查 | `npm i --no-audit` |
| **`--no-fund`** | 不显示依赖的资助信息 | `npm i --no-fund` |

### 🗂️ 路径与输出控制

| 参数 | 作用 | 示例 |
| :--- | :--- | :--- |
| **`--prefix <path>`** | 指定工作路径（相当于切换目录执行） | `npm i --prefix ./subproject` |
| **`--global-style`** | 强制依赖安装在顶层 `node_modules`，禁止扁平化 | `npm i --global-style` |
| **`--legacy-bundling`** | 使用 npm v4 之前的嵌套依赖结构（已很少用） | 略 |
| **`--loglevel <level>`** | 控制日志输出：`silent`, `error`, `warn`, `notice`, `http`, `info`, `verbose`, `silly` | `npm i --loglevel=verbose` |
| **`--silent`** | 等同于 `--loglevel=error` | `npm i --silent` |
| **`--quiet`** | 等同于 `--loglevel=warn` | `npm i --quiet` |
| **`--json`** | 以 JSON 格式输出结果（便于脚本解析） | `npm ls --json` |
| **`--no-progress`** | 关闭下载进度条 | `npm i --no-progress` |
| **`--no-color`** | 关闭彩色输出 | `npm i --no-color` |

### 🔐 安全与认证

| 参数 | 作用 |
| :--- | :--- |
| **`--otp <code>`** | 提供两步验证的动态码（发布、登录等操作） |
| **`--auth-type <web|legacy>`** | 登录时使用网页验证还是传统用户名密码 |
| **`--cert` / `--key`** | 客户端证书及私钥（私有 npm registry 需要） |

### 📦 部分命令特有参数举例

-   **`npm run <script> -- <args>`**  
    将参数传递给实际运行的脚本：  
    `npm run build -- --watch` → 实际执行 `npm run build --watch`

-   **`npm version patch --no-git-tag-version`**  
    升级版本但不打 git 标签。

-   **`npm publish --access public`**  
    将收费组织下的包发布为公共包。

-   **`npm install --install-links`** (较新)  
    将 workspace 中的本地 link 转换为真实 node_modules 副本。

### 🧪 快速查阅方法

-   **查看全部命令的参数**：`npm help <command>`（例如 `npm help install`）
-   **查看全局可用的配置**：`npm config ls -l`（会列出所有 npm 可识别的参数名，无论是否简写）
-   **常用组合**：  
    `npm install --no-save --no-package-lock --registry=https://my-registry.com`  
    `npm ci --production --ignore-scripts --no-audit --no-fund`

这些参数配合使用，可以精确控制 npm 在网络受限、CI环境、本地调试等场景下的行为。如果你有具体场景（比如公司内网、离线安装、加速CI），我可以给出更针对性的参数组合建议。
