好的，我来为你整理一份 **Git 常用命令大全**。我会按照使用场景和功能分类，尽量覆盖日常开发中最常用到的重要命令，而不是把所有命令都罗列出来（那样会非常长且不实用）。

> **说明**：下面列出的命令是经过筛选的“高频 + 实用”命令，而不是 `git --help` 里 100% 的完整列表。大厂日常工作中，90% 的场景只需要用到其中 30 条左右。

---

## 1️⃣ 初始化 & 配置

| 命令 | 说明 |
| :--- | :--- |
| `git init` | 在当前目录初始化一个新的 Git 仓库 |
| `git clone <url>` | 克隆远程仓库到本地 |
| `git config --global user.name "name"` | 设置全局用户名 |
| `git config --global user.email "email"` | 设置全局邮箱 |
| `git config --list` | 查看当前所有配置 |
| `git config --global core.editor "code --wait"` | 设置默认编辑器（示例：VS Code） |

---

## 2️⃣ 基本快照操作（日常提交）

| 命令 | 说明 |
| :--- | :--- |
| `git status` | 查看当前工作区和暂存区的状态（最常用） |
| `git add <file>` | 将指定文件添加到暂存区 |
| `git add .` | 添加当前目录下所有修改/新增的文件 |
| `git add -A` | 添加所有修改/新增/删除的文件（等价于 `git add .` 但更彻底） |
| `git commit -m "message"` | 提交暂存区内容，并附上提交信息 |
| `git commit -a -m "message"` | 跳过 `git add`，直接提交所有已跟踪文件的修改（不包括新文件） |
| `git commit --amend -m "new message"` | 修改最近一次提交的信息或内容 |
| `git rm <file>` | 从工作区和暂存区删除文件 |
| `git mv <old> <new>` | 移动或重命名文件（Git 会自动跟踪） |

---

## 3️⃣ 查看历史 & 差异

| 命令 | 说明 |
| :--- | :--- |
| `git log` | 查看提交历史 |
| `git log --oneline` | 单行简洁模式查看历史（最常用） |
| `git log --graph --oneline --decorate` | 图形化查看分支历史 |
| `git log -p <file>` | 查看某个文件的详细修改历史 |
| `git diff` | 查看工作区与暂存区的差异 |
| `git diff --staged` | 查看暂存区与上次提交的差异 |
| `git diff <commit1> <commit2>` | 比较两次提交之间的差异 |
| `git blame <file>` | 查看文件中每一行最后是谁修改的（定位问题用） |
| `git show <commit>` | 查看某次提交的详细信息 |

---

## 4️⃣ 分支管理

| 命令 | 说明 |
| :--- | :--- |
| `git branch` | 列出本地分支（当前分支前有 `*` 标记） |
| `git branch -r` | 列出远程分支 |
| `git branch -a` | 列出所有分支（本地 + 远程） |
| `git branch <branch-name>` | 创建新分支 |
| `git branch -d <branch-name>` | 删除本地分支（安全删除） |
| `git branch -D <branch-name>` | 强制删除未合并的分支 |
| `git checkout <branch-name>` | 切换分支 |
| `git checkout -b <branch-name>` | 创建并切换到新分支 |
| `git switch <branch-name>` | 切换分支（Git 2.23+ 推荐，语义更清晰） |
| `git switch -c <branch-name>` | 创建并切换（Git 2.23+） |
| `git merge <branch>` | 将指定分支合并到当前分支 |
| `git merge --abort` | 合并冲突后，放弃本次合并 |
| `git rebase <branch>` | 将当前分支变基到目标分支（线性历史） |
| `git rebase --continue` | 解决冲突后继续变基 |
| `git rebase --abort` | 放弃变基操作 |

---

## 5️⃣ 远程仓库同步

| 命令 | 说明 |
| :--- | :--- |
| `git remote -v` | 查看远程仓库地址 |
| `git remote add origin <url>` | 添加远程仓库（一般叫 origin） |
| `git remote remove origin` | 移除远程仓库 |
| `git push origin <branch>` | 将本地分支推送到远程 |
| `git push -u origin <branch>` | 推送并建立上游跟踪（以后可以直接 `git push`） |
| `git push origin --delete <branch>` | 删除远程分支 |
| `git push --tags` | 推送所有标签到远程 |
| `git pull` | 拉取远程当前分支的更新并合并（等价于 `fetch + merge`） |
| `git pull --rebase` | 拉取并用变基方式合并（推荐，避免多余的 merge 节点） |
| `git fetch` | 只拉取远程更新，不自动合并 |
| `git fetch --prune` | 拉取的同时清理远程已删除分支的本地跟踪 |

## 6️⃣ 撤销 & 重置

| 命令 | 说明 |
| :--- | :--- |
| `git restore <file>` | 丢弃工作区的修改（恢复成暂存区或最后一次提交的状态） |
| `git restore --staged <file>` | 将文件从暂存区撤出（等于 `git reset HEAD <file>`） |
| `git reset HEAD <file>` | 同上（旧语法） |
| `git reset --soft <commit>` | 撤销提交，但保留修改在暂存区 |
| `git reset --mixed <commit>` | 撤销提交和暂存，但保留修改在工作区（默认行为） |
| `git reset --hard <commit>` | 彻底回退到某个提交，丢弃所有后续修改（⚠️危险） |
| `git revert <commit>` | 通过创建一个新提交来“反向操作”某次提交（安全撤销） |
| `git clean -fd` | 删除未跟踪的文件和目录（⚠️不可恢复） |

---

## 7️⃣ 暂存 & 储藏

| 命令 | 说明 |
| :--- | :--- |
| `git stash` | 临时保存当前工作区的修改（用于切换分支） |
| `git stash save "message"` | 储藏并附带说明信息 |
| `git stash list` | 查看所有储藏记录 |
| `git stash pop` | 恢复最近一次储藏并从列表中删除 |
| `git stash apply` | 恢复最近一次储藏但保留在列表中 |
| `git stash drop` | 删除某个储藏 |
| `git stash clear` | 清空所有储藏（⚠️小心） |

---

## 8️⃣ 标签管理

| 命令 | 说明 |
| :--- | :--- |
| `git tag` | 列出所有标签 |
| `git tag <tagname>` | 创建轻量标签 |
| `git tag -a <tagname> -m "message"` | 创建附注标签（推荐） |
| `git show <tagname>` | 查看标签详情 |
| `git tag -d <tagname>` | 删除本地标签 |
| `git push origin <tagname>` | 推送单个标签到远程 |
| `git push --tags` | 推送所有标签 |
| `git push origin --delete <tagname>` | 删除远程标签 |

---

## 9️⃣ 高级 / 排查 / 工具类

| 命令 | 说明 |
| :--- | :--- |
| `git reflog` | 查看所有 HEAD 的历史移动（找回丢失的提交） |
| `git bisect start` | 开始二分查找定位引入 bug 的提交 |
| `git cherry-pick <commit>` | 将某个提交“摘取”到当前分支 |
| `git grep "pattern"` | 在代码库中搜索字符串（比系统 grep 快） |
| `git shortlog -sn` | 按提交数量排序查看贡献者 |
| `git archive --format=zip HEAD` | 导出当前代码为 zip 包 |
| `git rev-parse HEAD` | 获取当前提交的完整 SHA-1 哈希值 |
| `git ls-files` | 列出所有被 Git 跟踪的文件 |

| :--- | :--- | :--- |
| `git co` | `git checkout` | 切换分支 |
| `git br` | `git branch` | 分支列表 |
| `git ci` | `git commit` | 提交 |
| `git st` | `git status` | 查看状态 |
| `git lg` | `git log --oneline --graph --all` | 漂亮的图形化日志 |
