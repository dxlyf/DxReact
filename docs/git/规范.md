## Git 提交前缀完整清单

### 🎯 **核心类型（必须掌握）**

| 前缀 | 含义 | 说明 |
|------|------|------|
| **feat** | ✨ 新功能 | 新增特性、功能、页面等 |
| **fix** | 🐛 修复 | 修复 bug、错误 |
| **docs** | 📚 文档 | 仅修改文档（README、注释等） |
| **style** | 💎 样式 | 代码格式（空格、分号、缩进等），不影响逻辑 |
| **refactor** | ♻️ 重构 | 代码重构，不修 bug 不加功能 |
| **perf** | ⚡️ 性能 | 性能优化 |
| **test** | ✅ 测试 | 添加、修改测试用例 |
| **build** | 📦 构建 | 构建系统、依赖、打包工具等（webpack、vite、npm） |
| **ci** | 🎡 持续集成 | CI 配置（GitHub Actions、Jenkins、GitLab CI） |
| **chore** | 🔧 维护 | 其他杂项（更新依赖、配置文件等） |
| **revert** | ⏪ 回滚 | 回滚到之前的提交 |
| **merge** | 🔀 合并 | 合并分支 |

---

### 📋 **完整扩展类型**

| 前缀 | 含义 | 说明 |
|------|------|------|
| **types** | 📝 类型 | TypeScript 类型定义 |
| **release** | 🚀 发布 | 版本发布 |
| **deps** | 📦 依赖 | 依赖包更新 |
| **config** | ⚙️ 配置 | 配置文件修改 |
| **init** | 🎉 初始化 | 项目初始化 |
| **i18n** | 🌐 国际化 | 多语言、翻译 |
| **access** | ♿️ 可访问性 | 无障碍优化 |
| **assets** | 🍱 资源 | 图片、字体等静态资源 |
| **security** | 🔒 安全 | 安全修复 |
| **deprecate** | ⚠️ 弃用 | 标记废弃功能 |
| **remove** | 🔥 移除 | 删除代码或文件 |
| **rename** | ✏️ 重命名 | 重命名文件或变量 |
| **ui** | 💄 界面 | UI 样式调整 |
| **wip** | 🚧 进行中 | 部分完成，待完善 |

---

### 📝 **提交格式规范**

```
<类型>([可选作用域]): <描述>

[可选正文]

[可选脚注]
```

**示例：**
```bash
# 基本用法
feat: 添加用户登录功能

# 带作用域
feat(user): 添加头像上传功能

# 带破坏性变更
feat(api): 重构认证方式

BREAKING CHANGE: 改用 JWT 替代 Session

# 关联 Issue
fix: 修复支付超时问题

Closes #123, #456
```

### 🔥 **最常用 10 个（日常开发足够）**

```bash
1. feat:     新功能
2. fix:      修复 bug
3. docs:     修改文档
4. style:    格式化代码
5. refactor: 重构代码
6. perf:     性能优化
7. test:     测试相关
8. build:    构建相关
9. chore:    日常维护
10. merge:   合并分支
```

---

### 💡 **实战示例**

```bash
# 新功能
git commit -m "feat: 添加微信支付功能"

# 修复 bug
git commit -m "fix: 修复 iOS 下白屏问题"

# 文档
git commit -m "docs: 更新 API 文档"

# 样式
git commit -m "style: 格式化代码"

# 重构
git commit -m "refactor: 重构用户验证逻辑"

# 性能
git commit -m "perf: 虚拟列表优化"

# 测试
git commit -m "test: 添加单元测试"

# 构建
git commit -m "build: 升级 vue 3.3"

# 维护
git commit -m "chore: 更新依赖"

# 合并
git commit -m "merge: 合并 feature/login"

# 回滚
git commit -m "revert: 回滚提交 abc123"

# 国际化
git commit -m "i18n: 添加英文翻译"

# 移除代码
git commit -m "remove: 废弃旧 API"

# 类型定义
git commit -m "types: 完善用户类型定义"
```

### ✅ **推荐团队规范**

```
feat:      必须  新功能
fix:       必须  bug修复
docs:      可选  文档
style:     可选  格式化
refactor:  可选  重构
perf:      可选  性能
test:      可选  测试
build:     可选  构建
ci:        可选  CI
chore:     可选  维护
revert:    必须  回滚
merge:     必须  合并
```

**建议**：至少要求 `feat`、`fix`、`revert`、`merge` 这四个必须规范书写，其他可以根据团队情况灵活处理。
