# 🤖 AI Agent 辅助编程使用文档

## 一、核心原则

**把AI当作一个聪明但缺乏上下文的初级开发人员** —— 需要清晰指令、需求拆解和上下文信息。

---

## 二、通用提问框架（CRISP框架）

| 要素 | 说明 | 示例 |
|------|------|------|
| **C** - Context（上下文） | 项目类型、技术栈、现有代码 | "我在用React + TypeScript..." |
| **R** - Role（角色） | 让AI扮演什么角色 | "你是一个资深后端工程师" |
| **I** - Input（输入） | 提供具体输入/示例 | "这是现有代码：..." |
| **S** - Specification（规格） | 明确功能需求、输入输出 | "需要实现一个登录接口，接收username/password，返回token" |
| **P** - Preference（偏好） | 代码风格、注释要求等 | "使用async/await，添加错误处理" |

## 三、实战场景与提问技巧

### 🎯 场景1：从零开始创建一个功能/项目

**错误示范 ❌**
> "帮我写一个用户管理系统"

**正确示范 ✅**
> "帮我用Python + FastAPI实现一个用户管理API，需要包含：
> - 用户注册（用户名、邮箱、密码，密码需bcrypt加密）
> - 用户登录（返回JWT token）
> - 获取用户信息（需token认证）
> - 使用SQLite数据库 + SQLAlchemy ORM
> - 添加基础的错误处理和输入验证"

## 四、进阶技巧

### 1️⃣ 分步骤完成复杂任务

```
第一步：帮我设计一个待办事项App的数据库Schema（用户、任务、标签三张表）
第二步：基于这个Schema，生成创建表的SQL语句
第三步：用Node.js写一个连接数据库的模块
第四步：实现CRUD操作的API
```

### 2️⃣ 提供示例输入输出

```
实现一个函数 formatDuration(seconds)
输入：3661
输出："1小时1分1秒"

输入：120
输出："2分0秒"

输入：7200
输出："2小时0分0秒"

请实现边界情况处理（负数、超大数等）
```

### 3️⃣ 要求输出特定格式

```
请按以下格式回复：
## 方案概述
[一句话说明]

## 代码实现
```python
[完整代码]
```

## 使用示例
[演示如何使用]

## 依赖安装
[pip/ npm命令]

## 注意事项
[性能/安全/兼容性提醒]
```

### 4️⃣ 让AI写测试用例

```

测试应覆盖：
- 正常情况
- 边界值
- 异常输入
- 空值/None处理
```

### 5️⃣ 追加澄清和修正

```

2. 如果数据量很大（10万+），这个方案还适用吗？

```

## 七、常见陷阱与应对

| 问题 | 解决方案 |
|------|---------|
| AI给出过时代码 | 明确指定版本："Python 3.11 + FastAPI 0.100+" |
| 代码有安全漏洞 | 添加提示："请检查SQL注入、XSS、输入验证" |
| 忽略错误处理 | 明确要求："添加try-catch和错误日志" |
| 代码不可运行 | 要求："给出完整可运行的代码，包含所有import" |
| 没有注释 | 要求："添加中文注释解释关键逻辑" |

## 九、最佳实践速查

✅ **DO**
- 拆分大任务为小步骤
- 提供具体的输入输出示例
- 要求代码加上注释
- 验证AI给出的代码是否可运行
- 追问不理解的实现细节
- 要求提供多种解决方案并对比

❌ **DON'T**
- 不要假设AI知道你的项目上下文
- 不要一次性要求太多功能
- 不要在敏感代码中直接粘贴（脱敏）
- 不要盲目相信AI的代码（要测试）
- 不要期待AI理解业务语义（需要你解释）

## 十、示例对话全流程

**用户：**
> 我想用Python写一个脚本，监控指定文件夹的变化（新增、修改、删除文件），一旦有变化就把变化信息写入日志文件。请帮我实现。

**AI回复后...**

**用户跟进：**
> 你的实现用了watchdog库。我需要添加两个功能：
> 1. 忽略.tmp临时文件
> 2. 把日志按天分割（每天一个log文件）
> 请修改代码

**用户再跟进：**
> 如果我想改成只监控特定后缀的文件（比如 .py, .txt），怎么改？请只给出需要修改的部分。
