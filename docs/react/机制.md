React 19 的执行机制，本质上是一个**协同调度系统**。它不再被动地一气呵成完成渲染，而是主动地将渲染过程拆解、排序、暂停和恢复，确保用户的每一次点击和输入都能得到即时响应。

下面是整个机制的宏观鸟瞰图：

```mermaid
flowchart TD
    A[用户交互 / 状态更新] --> B[Scheduler 调度器]
    
    subgraph B [调度与优先级分配]
        direction LR

        B2 --> B3{ urgent <br>高优先级?}
    end

    B3 -- 是（如：输入框打字） --> C[进入“快车道”]
    B3 -- 否（如：搜索建议） --> D[进入“过渡道”<br>（Transition Lanes）]

    end

    subgraph D [并发可打断<br>（时间切片）]

        D2 -- 是 --> D3[暂停当前，处理高优任务]
        D3 --> D1
        D2 -- 否 --> D4[继续执行]
        D4 --> D1
    end

    D --> E
```

下面，我们深入到这张图中的每一个核心概念，看看它们具体是如何协作的。

### 🧬 Fiber：架构的基石
在 React 中，**Fiber 是虚拟 DOM 节点的“进化版”**。它是一个普通的 JavaScript 对象，但通过 `child`（第一个子节点）、`sibling`（兄弟节点）和 `return`（父节点）指针，将整个组件树连接成**链表结构**。

这种结构是**实现可中断渲染的前提**。因为它让 React 在完成一个节点的处理后，总能通过指针找到下一个要处理的节点，从而可以在任何时刻暂停和恢复工作。

### ⏳ Scheduler 与 Lane：工作的“交通系统”
这是 React 实现并发能力的“交通系统”，负责管理和调度所有更新任务。

*   **Scheduler（调度器）**：可以理解为一个**智能的交通指挥**。它决定什么任务可以立即执行（如用户输入），什么任务可以稍后执行（如网络请求结果渲染）。它通过**时间切片**（Time Slice）机制，将长任务切割成小块（每块约5ms），每执行完一块就主动“让出”主线程，检查是否有更紧急的任务需要处理，有效避免了页面卡顿。
*   **Lanes（车道）**：表示**任务的优先级**。React 内部维护了多个“车道”，如 `SyncLane`（最高优先级）、`InputContinuousLane`（用户交互）、`DefaultLane`（普通更新）、`TransitionLane`（低优先级过渡）等。高优先级的任务（如按钮点击）会“插队”，中断正在进行的低优先级任务。

作为开发者，我们主要通过两条“车道”与系统交互：**默认所有更新都是紧急的，只有主动将更新放入“过渡车道”（`startTransition`），它才会被标记为非紧急，让路给用户交互**。

### 📝 Update 与 Queue：变更的“待办清单”
当你调用 `setState` 时，并不会立即触发更新。React 会创建一个代表此次变更的 **Update 对象**（包含新数据等），并将其推入该 Fiber 节点的 **更新队列（Update Queue）** 中。

这个队列是应对频繁状态变更的关键。例如在短时间内连续多次调用 `setState`，React 会将对应的 Update 对象收集到队列中，在适当的时机进行**批量处理**，合并成一个更新，避免不必要的重复渲染。

### ⛳ Flags：DOM 操作的“施工清单”
如果说 Render 阶段是“计划”，那么 `flags` 就是这个计划的“施工清单”。它是一个**位掩码**（Bitmask），用于标记 Fiber 节点在 Commit 阶段需要执行的具体 DOM 操作。常见的 `flags` 包括：
*   `Placement`：代表需要**插入**或**移动** DOM 元素。
*   `Update`：代表需要**更新**组件的 Props、State 或 DOM 属性。
*   `ChildDeletion`：表示该 Fiber 节点需要**删除**某些子节点。
*   `Passive`：标记该组件包含 `useEffect`，需要在 Commit 的**被动阶段**执行。

### 🗺️ 两阶段渲染：从计划到落地
Fiber 架构最显著的特征，就是将整个渲染流程清晰地划分为两个阶段：**Render 阶段 和 Commit 阶段**。

*   **1. Render 阶段（可中断的“计划”阶段）**：这是**可中断、可异步**的。React 从根 Fiber 开始，向下遍历构建一棵新的 **“工作进行中”** Fiber 树（`workInProgress` tree）。在这个过程中，它会执行组件函数、进行 Diff 比对，并为发生变化的 Fiber 节点打上相应的 `flags` 标记。调度器会利用 Scheduler 和 Lane 在此阶段插入和中断任务。

*   **2. Commit 阶段（同步的“施工”阶段）**：这是**不可中断、同步**的。当一棵完整的 `workInProgress` 树构建完成后，React 会进入 Commit 阶段，根据 `flags` 一次性将所有的 DOM 变更同步应用到屏幕上。这个过程又分为三个子阶段：
    *   **Before Mutation**：在改变真实 DOM 前，可以读取 DOM 状态（如 `getSnapshotBeforeUpdate`）。
    *   **Mutation**：**执行 DOM 变更**。根据 `flags` 执行增、删、改操作。
    *   **Layout**：DOM 变更完成后、浏览器**即将绘制前**。这个阶段可以读取新的 DOM 布局，执行 `useLayoutEffect`。
    *   **Passive**：这是 `useEffect` 的执行阶段。它在浏览器**完成绘制之后**异步进行，因此不会阻塞屏幕的视觉更新。

### 💎 开发者核心收益：如何“指挥”这个系统
理解这套机制，最终是为了帮我们写出性能更好的 React 应用。核心在于，你不需要手动处理所有细节，只需用对 React 提供的工具：

*   **使用 `startTransition` 来标记非紧急更新**：对于输入框筛选大型列表的场景，用 `startTransition` 包裹列表更新逻辑，就能确保输入体验永远流畅。
*   **使用 `useDeferredValue` 来延迟处理渲染结果**：当你不方便控制状态来源时（比如接收来自父组件的 props），可以用 `useDeferredValue` 创建一个值的“滞后版本”，让昂贵的子组件渲染可以稍后、按需进行。
*   **注意渲染的“纯度”**：因为 Render 阶段可能会被中断、重启甚至丢弃，所以**组件函数（Root component function）必须保持纯粹**，不应该在此阶段执行任何副作用的操作（如网络请求、操作DOM），这些都应该放在 `useEffect` 或事件处理函数中。

希望这份拆解能帮你建立起 React 19 执行机制的整体图景。如果想深入了解某个具体概念，比如 `useTransition` 的最佳实践或 `useEffect` 的执行时机，随时可以再问我。

## 一、整体架构概览

React 19 延续了 Fiber 架构，但做了大量优化。核心流程分为两大阶段：

```
触发更新 → 创建/复用 Fiber → Schedule 调度 → Reconcile 调和 → Commit 提交
```

## 二、核心概念详解

### 1. Fiber（纤程/工作单元）

Fiber 是 React 的最小工作单元，本质上是一个**可中断、可恢复**的虚拟节点。

```javascript
// 简化的 Fiber 结构
interface Fiber {
  // 链表结构（构建树）
  child: Fiber | null;      // 第一个子节点
  sibling: Fiber | null;    // 下一个兄弟节点
  return: Fiber | null;     // 父节点
  
  // 双缓冲（Current vs WorkInProgress）
  alternate: Fiber | null;  // 当前树与工作中的树的对应节点
  
  // 类型与状态
  type: any;                // 组件类型（div, MyComponent）
  tag: WorkTag;             // 节点类型（HostComponent, FunctionComponent等）
  key: null | string;
  memoizedState: any;       // 当前状态（hooks链表头）
  memoizedProps: any;       // 当前props
  
  // 副作用标记
  flags: Flags;             // 自身副作用
  subtreeFlags: Flags;      // 子树副作用（聚合）
  
  // 更新队列
  updateQueue: UpdateQueue | null;
  
  // 优先级相关
  lanes: Lanes;             // 本节点待处理的更新车道
  childLanes: Lanes;        // 子树待处理的车道
  
  // DOM 引用
  stateNode: any;           // 真实DOM节点或组件实例
}
```

**双缓冲机制**：React 维护两棵树：
- **Current Tree**：当前屏幕上显示的树
- **WorkInProgress Tree**：正在构建的新树

渲染完成后，两棵树通过 `alternate` 互换，WorkInProgress 变成 Current。

### 2. Update（更新）

Update 是触发重新渲染的"指令"：

```javascript
interface Update<State> {
  lane: Lane;               // 优先级车道
  tag: UpdateTag;           // 更新类型（UpdateState, ReplaceState等）
  payload: any;             // 更新载荷（新state或reducer）
  callback: Callback | null; // setState的第二个参数
  
  next: Update | null;      // 环形链表，指向下一个update
}

interface UpdateQueue<State> {
  baseState: State;         // 基础状态
  firstBaseUpdate: Update | null;  // 上次未完成的更新链表头
  lastBaseUpdate: Update | null;   // 上次未完成的更新链表尾
  shared: {
    pending: Update | null; // 新加入的更新（环形链表）
  };
  effects: Array<Callback> | null; // 副作用回调
}
```

**为什么用环形链表？** 方便在链表尾部快速添加新 Update，同时可以从任意节点遍历整个链表。

### 3. Queue（队列）

React 19 中的 Queue 不仅用于 state，还用于：

- **State Queue**：`useState`/`useReducer` 的更新队列
- **Effect Queue**：`useEffect`/`useLayoutEffect` 的副作用队列
- **Transition Queue**：`useTransition` 的过渡队列

**批量更新（Batching）**：React 19 默认自动批处理所有更新（包括 setTimeout、Promise 等），多个 setState 合并为一次重渲染。

### 4. Lanes（车道模型）

Lanes 是 React 18+ 的**优先级系统**，用二进制位表示：

```javascript
// Lane 是单个位，Lanes 是多个位的组合
type Lane = number;   // 单个车道  0b0000000000000000000000000000001
type Lanes = number;  // 车道集合  0b0000000000000000000000000000011

// 优先级从高到低
export const SyncLane: Lane = /*                        */ 0b0000000000000000000000000000001; // 同步，最高
export const InputContinuousHydrationLane: Lane = /*    */ 0b0000000000000000000000000000010;
export const InputContinuousLane: Lane = /*             */ 0b0000000000000000000000000000100;
export const DefaultHydrationLane: Lane = /*            */ 0b0000000000000000000000000001000;
export const DefaultLane: Lane = /*                     */ 0b0000000000000000000000000010000; // 默认
export const TransitionHydrationLane: Lane = /*         */ 0b0000000000000000000000000100000;
export const TransitionLanes: Lanes = /*                */ 0b0000000001111111111111111000000; // 过渡（共16条）
export const RetryLanes: Lanes = /*                     */ 0b0000111110000000000000000000000;
export const IdleLane: Lane = /*                        */ 0b0010000000000000000000000000000; // 空闲
export const OffscreenLane: Lane = /*                   */ 0b0100000000000000000000000000000;
```

**车道模型的优势**：
- **位运算高效**：合并、提取、比较都是 O(1)
- **支持批量**：一个更新可以属于多个车道
- **支持饥饿预防**：低优先级任务不会永远被高优先级阻塞

### 5. Flags & SubtreeFlags（副作用标记）

```javascript
// 副作用类型（节选）
export const Placement = /*             */ 0b0000000000000000000000000000010; // 插入DOM
export const Update = /*                */ 0b0000000000000000000000000000100; // 更新DOM
export const Deletion = /*              */ 0b0000000000000000000000000001000; // 删除DOM
export const ChildDeletion = /*         */ 0b0000000000000000000000010000000; // 子节点删除
export const Ref = /*                   */ 0b0000000000000000000000100000000; // ref更新
export const Passive = /*               */ 0b0000000000000000000001000000000; // useEffect
export const LayoutMask = /*            */ 0b0000000000000000000010000000000; // useLayoutEffect
export const Visibility = /*            */ 0b0000000000000000000100000000000; // 可见性变化
export const MountLayoutDev = /*        */ 0b0000000000000000010000000000000;
export const MountPassiveDev = /*       */ 0b0000000000000000100000000000000;
```

**SubtreeFlags 的作用**：
- 在 Reconcile 阶段，子节点的副作用会**冒泡**到父节点的 `subtreeFlags`
- Commit 阶段可以通过检查 `subtreeFlags` 快速判断是否需要遍历子树，避免不必要的遍历

```javascript
// 冒泡逻辑（简化）
function bubbleProperties(completedWork: Fiber) {
  let subtreeFlags = NoFlags;
  let child = completedWork.child;
  while (child !== null) {
    subtreeFlags |= child.subtreeFlags | child.flags;
    child = child.sibling;
  }
  completedWork.subtreeFlags |= subtreeFlags;
}
```

### 6. Time Slice（时间切片）

Time Slice 是实现**可中断渲染**的基础：

```javascript
// 每帧留给 React 的工作时间（通常 5ms）
const frameInterval = 5;

function shouldYield(): boolean {
  const timeElapsed = getCurrentTime() - startTime;
  return timeElapsed >= frameInterval; // 超过5ms就中断
}

// 工作循环
function workLoopConcurrent() {
  while (workInProgress !== null && !shouldYield()) {
    performUnitOfWork(workInProgress); // 执行一个 Fiber 单元
  }
}
```

**为什么需要 5ms？** 为了保证 60fps 的流畅度，一帧 16.6ms，React 只占用一部分，留出时间给浏览器处理用户输入、动画等。

### 7. Scheduler（调度器）

React 19 使用独立的 Scheduler 包，核心是一个**基于优先级队列**的任务调度系统：

```javascript
// 任务优先级（与 Lanes 对应但独立）
export const NoPriority = 0;
export const ImmediatePriority = 1;   // 同步执行
export const UserBlockingPriority = 2; // 用户交互（250ms超时）
export const NormalPriority = 3;      // 普通（5s超时）
export const LowPriority = 4;         // 低优先级（10s超时）
export const IdlePriority = 5;        // 空闲（无超时）

// 任务结构
interface Task {
  id: number;
  callback: Function | null;
  priorityLevel: PriorityLevel;
  startTime: number;      // 开始时间
  expirationTime: number; // 过期时间（startTime + timeout）
  sortIndex: number;      // 排序依据（过期时间或开始时间）
}
```

**调度策略**：
1. **MessageChannel**（优先）：宏任务，不阻塞渲染
2. `setImmediate`（Node）
3. `setTimeout`（降级）

**时间切片实现**：
```javascript
// 使用 MessageChannel 实现宏任务调度
const channel = new MessageChannel();
const port = channel.port2;

// 请求调度
schedulePerformWorkUntilDeadline() {
  port.postMessage(null); // 发送消息触发回调
}

// 在回调中执行工作
channel.port1.onmessage = performWorkUntilDeadline;
```

### 8. Concurrent（并发模式）

Concurrent 是 React 18+ 的核心特性，允许**可中断、可恢复的渲染**：

```javascript
// React 18 之前（Legacy Mode）
ReactDOM.render(<App />, root); // 同步，不可中断

// React 18+（Concurrent Mode）
const root = ReactDOM.createRoot(rootElement);
root.render(<App />); // 并发，可中断
```

**并发特性**：
- **可中断渲染**：高优先级更新可以打断低优先级渲染
- **自动批处理**：所有事件中的 setState 自动合并
- **Suspense**：真正的异步组件支持
- **Transitions**：`useTransition` 标记非紧急更新

### 9. ReconcileChildren（调和子节点）

这是 Diff 算法的核心，React 19 采用**双端比较**优化：

```javascript
function reconcileChildren(
  current: Fiber | null,
  workInProgress: Fiber,
  nextChildren: any,
  renderLanes: Lanes
) {
  if (current === null) {
    // 首次挂载：创建新的 Fiber
    workInProgress.child = mountChildFibers(
      workInProgress,
      null,
      nextChildren,
      renderLanes
    );
  } else {
    // 更新：对比 diff
    workInProgress.child = reconcileChildFibers(
      workInProgress,
      current.child,
      nextChildren,
      renderLanes
    );
  }
}
```

**Diff 策略（单链表，O(n)）**：

1. **第一轮**：按顺序比较，key 和 type 相同则复用
   ```
   旧: A - B - C - D
   新: A - B - E - C
   复用 A、B，C 和 D 放入 map
   ```

2. **第二轮**：处理剩余节点
   - 新节点在 map 中找到则复用并移动
   - 找不到则创建新 Fiber

3. **移动优化**：通过 `lastPlacedIndex` 判断是否需要移动

```javascript
// 判断是否需要移动
if (oldIndex < lastPlacedIndex) {
  // 需要移动（标记 Placement）
  newFiber.flags |= Placement;
} else {
  lastPlacedIndex = oldIndex; // 更新最后位置
}
```

**React 19 的优化**：对于简单列表（无 key 变化），采用更轻量的比较策略。

## 三、完整执行流程

### 1. 触发更新阶段

```javascript
// 以 setState 为例
function dispatchSetState(fiber, queue, action) {
  // 1. 创建 Update
  const update = createUpdate(lane);
  update.payload = { lane, action };
  
  // 2. 加入队列（环形链表）
  enqueueUpdate(fiber, queue, update);
  
  // 3. 调度更新
  const root = scheduleUpdateOnFiber(fiber, lane, eventTime);
}
```

### 2. Schedule 阶段

```javascript
function scheduleUpdateOnFiber(fiber, lane, eventTime) {
  // 1. 标记更新车道
  markRootUpdated(root, lane, eventTime);
  
  // 2. 判断是否需要调度
  if (lane === SyncLane) {
    // 同步更新：立即执行
    performSyncWorkOnRoot(root);
  } else {
    // 并发更新：交给 Scheduler
    ensureRootIsScheduled(root, eventTime);
  }
}

function ensureRootIsScheduled(root) {
  const nextLanes = getNextLanes(root, NoLanes);
  
  // 根据优先级选择调度方式
  if (nextLanes === SyncLane) {
    // 同步：微任务或立即执行
    scheduleSyncCallback(performSyncWorkOnRoot.bind(null, root));
  } else {
    // 并发：通过 Scheduler 调度
    scheduleCallback(priority, performConcurrentWorkOnRoot.bind(null, root));
  }
}
```

### 3. Render 阶段（Reconcile + Time Slice）

```javascript
function performConcurrentWorkOnRoot(root, didTimeout) {
  // 1. 获取要处理的车道
  const lanes = getNextLanes(root, NoLanes);
  
  // 2. 开始工作循环（可中断）
  let exitStatus = renderRootConcurrent(root, lanes);
  
  // 3. 如果完成，进入 Commit
  if (exitStatus === RootCompleted) {
    const finishedWork = root.current.alternate;
    root.finishedWork = finishedWork;
    root.finishedLanes = lanes;
    finishConcurrentRender(root, exitStatus, lanes);
  }
}

function renderRootConcurrent(root, lanes) {
  do {
    try {
      workLoopConcurrent(); // 时间切片循环
      break;
    } catch (thrownValue) {
      handleError(root, thrownValue);
    }
  } while (true);
  
  return workInProgressRootExitStatus;
}

function workLoopConcurrent() {
  while (workInProgress !== null && !shouldYield()) {
    performUnitOfWork(workInProgress);
  }
}
```

**performUnitOfWork 深度优先遍历**：
```javascript
function performUnitOfWork(unitOfWork: Fiber): void {
  const current = unitOfWork.alternate;
  
  // 1. 开始工作（beginWork）：创建/复用子 Fiber
  let next = beginWork(current, unitOfWork, renderLanes);
  
  if (next === null) {
    // 2. 没有子节点，完成工作（completeWork）
    completeUnitOfWork(unitOfWork);
  } else {
    // 3. 继续处理子节点
    workInProgress = next;
  }
}

function completeUnitOfWork(unitOfWork: Fiber): void {
  let completedWork = unitOfWork;
  do {
    // 1. 完成当前节点（生成 DOM、标记副作用）
    const next = completeWork(current, completedWork, renderLanes);
    
    // 2. 冒泡 subtreeFlags
    bubbleProperties(completedWork);
    
    // 3. 处理兄弟节点
    const siblingFiber = completedWork.sibling;
    if (siblingFiber !== null) {
      workInProgress = siblingFiber;
      return;
    }
    
    // 4. 返回父节点
    completedWork = completedWork.return;
  } while (completedWork !== null);
}
```

### 4. Commit 阶段（不可中断）

Commit 分为三个子阶段，**同步执行**：

```javascript
function commitRoot(root) {
  const { finishedWork, finishedLanes } = root;
  
  // 1. Before Mutation（DOM 操作前）
  commitBeforeMutationEffects(finishedWork);
  //    - 读取 DOM 状态（如 scroll）
  //    - 调用 getSnapshotBeforeUpdate
  
  // 2. Mutation（DOM 操作）
  commitMutationEffects(root, renderPriorityLevel, finishedWork);
  //    - 插入、更新、删除 DOM
  //    - 重置文本内容
  
  // 3. Root 切换（双缓冲交换）
  root.current = finishedWork;
  
  // 4. Layout（DOM 操作后，同步执行）
  commitLayoutEffects(finishedWork, root, lanes);
  //    - 调用 componentDidMount/Update
  //    - 调用 useLayoutEffect
  //    - 设置 ref
  
  // 5. Passive Effects（异步执行）
  schedulePassiveEffects(finishedWork);
  //    - 调用 useEffect（下一个宏任务）
}
```

## 四、React 19 的新特性与优化

### 1. Compiler（React Compiler）
React 19 引入了自动记忆化编译器，在构建时自动添加 `useMemo`/`useCallback` 等价优化，减少运行时开销。

### 2. Actions & Transitions
```javascript
// 自动处理 pending 状态
function UpdateName() {
  const [name, setName] = useState("");
  
  // 自动标记为 Transition
  const handleSubmit = async () => {
    await updateName(name); // Server Action
  };
  
  return <form action={handleSubmit}>...</form>;
}
```

### 3. 改进的 Suspense
- **不瀑布化**：兄弟 Suspense 组件并行获取数据
- **更精确的边界**：支持更细粒度的加载状态

### 4. RSC（React Server Components）
服务端组件不进入 Fiber 树，直接渲染为特殊标记，客户端 React 将其与 Client Components 合并。

## 五、总结图

```
┌─────────────────────────────────────────────────────────────┐
│                        触发更新                               │
│  setState / forceUpdate / dispatch / useReducer / etc.       │
└──────────────────────┬──────────────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                      Update Queue                            │
│  环形链表存储更新，支持批量合并（Batching）                     │
└──────────────────────┬──────────────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                     Schedule 调度                            │
│  1. Lane 优先级计算 → 2. 选择调度策略（Sync/Concurrent）      │
│  3. Scheduler 优先级队列 → 4. MessageChannel 宏任务调度       │
└──────────────────────┬──────────────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                     Render 阶段（可中断）                      │
│  beginWork: 创建/复用 Fiber（reconcileChildren - Diff算法）   │
│  completeWork: 生成 DOM，冒泡 Flags/SubtreeFlags              │
│  时间切片: shouldYield() 每 5ms 检查是否让出主线程             │
└──────────────────────┬──────────────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                     Commit 阶段（同步）                        │
│  BeforeMutation → Mutation（DOM操作）→ Layout（同步Effect）   │
│  → Passive（异步useEffect）                                   │
└─────────────────────────────────────────────────────────────┘
```

React 19 的核心设计哲学是：**将工作拆分为最小单元（Fiber），通过优先级（Lanes）和调度器（Scheduler）实现可中断的并发渲染，通过双缓冲和副作用标记（Flags）实现高效的 DOM 更新**。
