# Vue3 vs React 路由守卫设计对比

## 一、核心差异概览

| 维度 | **Vue3 (Vue Router 4)** | **React (React Router v6)** |
|------|------------------------|----------------------------|
| **守卫位置** | 路由配置文件中（声明式） | 组件内部（命令式） |
| **守卫类型** | 全局前置/后置、路由独享、组件内 | 无内置守卫，需自定义组件 |
| **执行时机** | 路由跳转前自动触发 | 渲染阶段判断，组件加载前 |
| **中断跳转** | `return false` / `next(false)` | 渲染 `<Navigate>` 组件 |
| **权限获取** | 同步/异步均可 | 通常通过 Context/Store |
| **代码组织** | 集中式（路由配置 + 守卫函数） | 分散式（守卫组件包裹路由） |

## 二、Vue3 路由守卫实现

### 1️⃣ 基础示例

```javascript
// router/index.js
import { createRouter, createWebHistory } from 'vue-router'
import { useAuthStore } from '@/stores/auth'

const routes = [
  {
    path: '/login',
    name: 'Login',
    component: () => import('@/views/Login.vue'),
    meta: { requiresGuest: true }  // 仅未登录可访问
  },
  {
    path: '/dashboard',
    name: 'Dashboard',
    component: () => import('@/views/Dashboard.vue'),
    meta: { requiresAuth: true }   // 需要登录
  },
  {
    path: '/admin',
    name: 'Admin',
    component: () => import('@/views/Admin.vue'),
    meta: { requiresAuth: true, role: 'admin' }
  }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

// 全局前置守卫（核心）
router.beforeEach(async (to, from, next) => {
  const authStore = useAuthStore()
  
  // 等待异步状态恢复（如刷新页面后恢复登录态）
  await authStore.restoreSession()
  
  // 1. 需要登录但未登录
  if (to.meta.requiresAuth && !authStore.isAuthenticated) {
    next({
      path: '/login',
      query: { redirect: to.fullPath }  // 记住目标路径
    })
    return
  }
  
  // 2. 已登录但访问游客页面（如登录页）
  if (to.meta.requiresGuest && authStore.isAuthenticated) {
    next({ path: '/dashboard' })
    return
  }
  
  // 3. 角色权限校验
  if (to.meta.role && authStore.user?.role !== to.meta.role) {
    next({ path: '/403' })
    return
  }
  
  // 4. 通过检查
  next()
})

// 全局后置钩子（可选）
router.afterEach((to, from) => {
  // 页面访问统计、修改标题等
  document.title = to.meta.title || '默认标题'
})

export default router
```

### 3️⃣ 组件内守卫

```vue
<template>
  <div>用户详情页</div>
</template>

<script setup>
import { onBeforeRouteLeave, onBeforeRouteUpdate } from 'vue-router'

// 离开前检查是否有未保存内容
onBeforeRouteLeave((to, from, next) => {
  if (formChanged.value) {
    const answer = confirm('有未保存的更改，确定要离开吗？')
    next(answer)
  } else {
    next()
  }
})

// 同一组件参数变化时（如 /user/1 -> /user/2）
onBeforeRouteUpdate(async (to, from, next) => {
  if (to.params.id !== from.params.id) {
    await fetchUserData(to.params.id)
  }
  next()
})
</script>
```

## 三、React 路由守卫实现（对比参考）

### 1️⃣ 自定义守卫组件（最常见）

```jsx
// components/PrivateRoute.jsx
import { Navigate, useLocation } from 'react-router-dom'
import { useAuth } from '@/hooks/useAuth'

export const PrivateRoute = ({ children, requiredRole }) => {
  const { isAuthenticated, user, loading } = useAuth()
  const location = useLocation()
  
  if (loading) return <div>Loading...</div>
  
  if (!isAuthenticated) {
    return <Navigate to="/login" state={{ from: location }} replace />
  }
  
  if (requiredRole && user?.role !== requiredRole) {
    return <Navigate to="/403" replace />
  }
  
  return children
}

// 使用示例
<Route 
  path="/admin" 
  element={
    <PrivateRoute requiredRole="admin">
      <Admin />
    </PrivateRoute>
  } 
/>
```

### 2️⃣ 守卫 Hook 封装（提高复用性）

```jsx
// hooks/useRouteGuard.js
import { useEffect } from 'react'
import { useNavigate, useLocation } from 'react-router-dom'
import { useAuth } from './useAuth'

export const useRouteGuard = (config) => {
  const { isAuthenticated, user, loading } = useAuth()
  const navigate = useNavigate()
  const location = useLocation()
  
  useEffect(() => {
    if (loading) return
    
    // 需要登录但未登录
    if (config.requiresAuth && !isAuthenticated) {
      navigate('/login', { state: { from: location }, replace: true })
      return
    }
    
    // 已登录但访问游客页面
    if (config.requiresGuest && isAuthenticated) {
      navigate('/dashboard', { replace: true })
      return
    }
    
    // 角色校验
    if (config.requiredRole && user?.role !== config.requiredRole) {
      navigate('/403', { replace: true })
    }
  }, [loading, isAuthenticated, user, location, navigate, config])
}

// 在组件中使用
function Dashboard() {
  useRouteGuard({ requiresAuth: true })
  return <div>Dashboard</div>
}
```

### 3️⃣ 统一守卫布局（推荐用于大型应用）

```jsx
// layouts/ProtectedLayout.jsx
import { Outlet } from 'react-router-dom'
import { useAuth } from '@/hooks/useAuth'
import { PermissionProvider } from '@/contexts/PermissionContext'

export const ProtectedLayout = () => {
  const { isAuthenticated, loading } = useAuth()
  
  if (loading) return <LoadingSpinner />
  if (!isAuthenticated) return <Navigate to="/login" />
  
  return (
    <PermissionProvider>
      <div className="app-layout">
        <Sidebar />
        <main>
          <Outlet /> {/* 所有子路由在此渲染 */}
        </main>
      </div>
    </PermissionProvider>
  )
}

// 路由配置
<Routes>
  <Route element={<ProtectedLayout />}>
    <Route path="/dashboard" element={<Dashboard />} />
    <Route path="/profile" element={<Profile />} />
    <Route path="/settings" element={<Settings />} />
  </Route>
</Routes>
```

## 四、设计对比与最佳实践

### 🎯 Vue3 设计哲学：集中式配置

**优点：**
- 所有路由逻辑集中管理，易于维护
- 路由元信息（`meta`）非常灵活
- 原生支持异步守卫，代码清晰

**最佳实践：**

```javascript
// 1. 分离守卫逻辑到独立模块
// guards/authGuard.js
export const authGuard = async (to, from, next) => {
  // 复杂校验逻辑
}

// router/index.js
import { authGuard } from './guards/authGuard'
router.beforeEach(authGuard)

// 2. 使用枚举定义权限级别
// constants/roles.js
export const ROLES = {
  ADMIN: 'admin',
  USER: 'user',
  GUEST: 'guest'
}

// 3. 封装权限指令（配合组件级权限）
// directives/permission.js
app.directive('permission', {
  mounted(el, binding) {
    const hasPermission = checkPermission(binding.value)
    if (!hasPermission) el.remove()
  }
})
```

### ⚛️ React 设计哲学：组合式组件

**优点：**
- 符合 React 组件化思维
- 与 Context/Hooks 配合自然
- 更灵活，可定制性强

**最佳实践：**

```jsx
// 1. 创建守卫工厂函数
// factories/createGuard.jsx
export const createGuard = (condition, redirectPath) => {
  return ({ children, ...props }) => {
    const shouldRedirect = condition(props)
    if (shouldRedirect) return <Navigate to={redirectPath} />
    return children
  }
}

// 使用
const AdminGuard = createGuard(
  (props) => props.userRole !== 'admin',
  '/403'
)

// 2. 守卫组合器
const composeGuards = (...guards) => ({ children, ...props }) => {
  let element = children
  for (const Guard of guards.reverse()) {
    element = <Guard {...props}>{element}</Guard>
  }
  return element
}

// 使用
const FullGuard = composeGuards(AuthGuard, RoleGuard, RateLimitGuard)

// 3. 路由配置中心化（虽然守卫在组件内，但配置可集中）
// routes.config.js
export const routes = [
  {
    path: '/dashboard',
    component: Dashboard,
    guards: ['auth', 'role:user']
  }
]

// 动态生成路由（配合守卫映射）
const guardMap = {
  auth: AuthGuard,
  role: RoleGuard
}
```

## 五、核心设计决策指南

| 场景 | **推荐方案** | **理由** |
|------|-------------|---------|
| **中小型项目** | Vue: 全局前置守卫<br>React: 守卫组件包裹 | 简单直观，代码量少 |
| **大型企业应用** | Vue: 守卫 + 权限指令<br>React: 守卫布局 + HOC | 可维护性强，复用性好 |
| **复杂权限模型** | Vue: meta + 全局守卫<br>React: Context + 守卫组合器 | 灵活配置，易于扩展 |
| **微前端架构** | Vue: 路由独享守卫<br>React: 守卫 Hook | 避免全局污染 |
| **SSR场景** | Vue: `beforeEach` + `onServerPrefetch`<br>React: 服务端守卫逻辑 + 客户端唤醒 | 注意异步数据同步 |

## 七、总结表格

| 维度 | **Vue3 胜出项** | **React 胜出项** |
|------|----------------|-----------------|
| **上手难度** | ✅ 官方内置，开箱即用 | ❌ 需要手动实现 |
| **灵活性** | ❌ 相对受限 | ✅ 极高的组合自由度 |
| **代码组织** | ✅ 集中式，易维护 | ❌ 分散在各组件 |
| **TypeScript 支持** | ✅ 官方类型完善 | ✅ 类型推断灵活 |
| **性能** | ✅ 守卫在路由层执行 | ❌ 组件可能多余渲染 |
| **测试友好** | ❌ 守卫测试较复杂 | ✅ 守卫组件易于单元测试 |

**最终建议：**
- 如果追求**开箱即用**和**约定大于配置** → 选择 **Vue3** 的路由守卫设计
- 如果需要**极致灵活**和**高度自定义** → 选择 **React** 的组合式守卫方案

两种框架都能实现完善的权限控制，核心差异在于**设计哲学的取舍**。你需要根据团队偏好和项目规模来权衡。
