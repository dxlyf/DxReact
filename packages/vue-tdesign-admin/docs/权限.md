在 React Router v6 中，路由守卫的实现思路很清晰：**创建一个守卫组件（Wrapper Component）来包裹需要保护的路由**，在这个组件里进行权限判断，再决定是渲染子组件还是重定向到其他页面。

下面是基于 v6 版本的标准做法。

---

### 🛡️ 1. 基础登录守卫：`PrivateRoute`

这个组件用于保护需要用户登录才能访问的页面（比如个人仪表盘）。如果未登录，用户会被重定向到登录页。

```jsx
// src/components/PrivateRoute.jsx
import { Navigate, useLocation } from "react-router-dom";

/**
 * 登录路由守卫
 * @param {Object} props
 * @param {boolean} props.isAuthenticated - 用户是否已登录
 * @param {React.ReactNode} props.children - 要保护的目标页面组件
 */
export const PrivateRoute = ({ isAuthenticated, children }) => {
  const location = useLocation();

  if (!isAuthenticated) {
    // 未登录时重定向到登录页，并携带当前路径，方便登录后跳回
    return <Navigate to="/login" state={{ from: location }} replace />;
  }

  // 已登录则正常渲染目标页面
  return children;
};
```

> **注意**：`replace` 属性可以防止未登录用户通过浏览器的“后退”按钮，再次回到受限制的页面。

### 🚫 2. 反向守卫：`AnonymousRoute`（或 `PublicRoute`）

这个守卫用于登录后的用户场景——当用户已经登录后，不应该再让他访问 `/login` 或 `/register` 这类页面，此时应将其重定向到首页或仪表盘。

```jsx
// src/components/AnonymousRoute.jsx
import { Navigate } from "react-router-dom";

/**
 * 反向路由守卫：仅允许未登录用户访问（如登录页、注册页）
 * @param {Object} props
 * @param {boolean} props.isAuthenticated - 用户是否已登录
 * @param {React.ReactNode} props.children - 要保护的页面组件
 */
export const AnonymousRoute = ({ isAuthenticated, children }) => {
  if (isAuthenticated) {
    // 如果用户已经登录，他想去登录页，直接把他送去仪表盘
    return <Navigate to="/dashboard" replace />;
  }

  return children;
};
```

### 🔗 3. 在路由配置中使用（`App.jsx`）

在 React Router v6 中，守卫的使用方式不再是 `<PrivateRoute path="..." />`，而是将目标页面**作为 `children` 传递给守卫组件**，然后赋值给 `<Route>` 的 `element` 属性。

```jsx
// src/App.jsx
import { BrowserRouter, Routes, Route } from "react-router-dom";
import { PrivateRoute } from "./components/PrivateRoute";
import { AnonymousRoute } from "./components/AnonymousRoute";
// 假设这些页面组件已存在
import Dashboard from "./pages/Dashboard";
import Login from "./pages/Login";
import Home from "./pages/Home";

// 模拟一个全局的认证状态 Hook
const useAuth = () => {
  // 这里通常会从 Context 或 localStorage 读取用户信息
  const user = localStorage.getItem("user");
  return { isAuthenticated: !!user };
};

function App() {
  const { isAuthenticated } = useAuth();

  return (
    <BrowserRouter>
      <Routes>
        {/* 公开页面，所有人都可以访问 */}
        <Route path="/" element={<Home />} />

        {/* 登录页：只有未登录用户才能访问 */}
        <Route
          path="/login"
          element={
            <AnonymousRoute isAuthenticated={isAuthenticated}>
              <Login />
            </AnonymousRoute>
          }
        />

        {/* 仪表盘：受保护的页面，必须登录才能看 */}
        <Route
          path="/dashboard"
          element={
            <PrivateRoute isAuthenticated={isAuthenticated}>
              <Dashboard />
            </PrivateRoute>
          }
        />
      </Routes>
    </BrowserRouter>
  );
}

export default App;
```

### 🧩 4. 进阶用法：嵌套布局统一保护

如果你的应用有多个受保护的页面（例如 `/dashboard`、`/profile`、`/settings`），每次写 `<PrivateRoute>` 会有点啰嗦。利用 v6 的嵌套路由特性，可以创建一个布局组件，只守卫一次：

```jsx
// 1. 创建一个受保护的布局组件
import { Outlet } from "react-router-dom";
import { PrivateRoute } from "./PrivateRoute";

export const ProtectedLayout = ({ isAuthenticated }) => {
  return (
    <PrivateRoute isAuthenticated={isAuthenticated}>
      {/* 这里可以放顶栏、侧边栏等共享布局 */}
      <div className="app-container">
        <Outlet /> {/* 子路由会在这里渲染 */}
      </div>
    </PrivateRoute>
  );
};

// 2. 在路由配置中使用 Layout
// src/App.jsx
// ...
<Route
  element={<ProtectedLayout isAuthenticated={isAuthenticated} />}
>
  {/* 所有这些子路由都会受到 ProtectedLayout 的保护 */}
  <Route path="/dashboard" element={<Dashboard />} />
  <Route path="/profile" element={<Profile />} />
  <Route path="/settings" element={<Settings />} />
</Route>
```

### 🧠 5. 与 Context 结合（生产环境推荐）

实际项目中，认证状态通常通过 React Context 或 Redux 来管理，而不是直接读 `localStorage` 并逐层传递 `isAuthenticated`。下面是一个整合了 Context 的完整示例：

```jsx
// src/context/AuthContext.jsx
import { createContext, useContext, useState, useEffect } from "react";

const AuthContext = createContext(null);

export const AuthProvider = ({ children }) => {
  const [isAuthenticated, setIsAuthenticated] = useState(false);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    // 模拟从服务器验证 Token
    const checkAuth = async () => {
      const token = localStorage.getItem("token");
      // 这里调用 API 验证 token 有效性
      setIsAuthenticated(!!token);
      setLoading(false);
    };
    checkAuth();
  }, []);

  const login = (token) => {
    localStorage.setItem("token", token);
    setIsAuthenticated(true);
  };

  const logout = () => {
    localStorage.removeItem("token");
    setIsAuthenticated(false);
  };

  return (
    <AuthContext.Provider value={{ isAuthenticated, loading, login, logout }}>
      {children}
    </AuthContext.Provider>
  );
};

export const useAuth = () => useContext(AuthContext);
```

修改守卫组件，从 Context 获取状态：

```jsx
// src/components/PrivateRoute.jsx
import { Navigate } from "react-router-dom";
import { useAuth } from "../context/AuthContext";

export const PrivateRoute = ({ children }) => {
  const { isAuthenticated, loading } = useAuth();

  if (loading) {
    return <div>Loading...</div>; // 或加载骨架屏，避免闪烁
  }

  return isAuthenticated ? children : <Navigate to="/login" replace />;
};
```

### ✅ 小结

| 场景 | 解决方案 | 关键重定向 |
| :--- | :--- | :--- |
| **未登录访问个人中心** | `PrivateRoute` | 跳转到 `/login` |
| **已登录访问登录页** | `AnonymousRoute` | 跳转到 `/dashboard` |
| **多个页面需要统一保护** | 嵌套路由 + 守卫布局 | 只需守卫布局一次 |

这种“组合式”的守卫方案完全符合 React Router v6 的设计理念，清晰且可复用。

如果你需要实现更复杂的场景，比如基于用户角色的权限控制（如只有 `admin` 角色才能访问管理页面），或者想了解异步权限校验的最佳实践，随时告诉我～
